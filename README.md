# Filament & Form

A 3D-printing portfolio site built around the Creality K2 Plus. Showcase artwork, drop viral-print files for the community, and turn admirers into conversations on WhatsApp or Instagram — not into checkout traffic.

Built with:

- **Next.js 15** (App Router) + **React 19** + **TypeScript**
- **Tailwind v4** + **shadcn/ui** + **motion** (Framer Motion) + **lenis** smooth-scroll
- **Payload CMS 3** (self-hosted, runs inside the same Next app) with **SQLite** for local dev

## Phase 1 includes

- Animated, full-bleed hero with a "filament" palette and live build specs
- Marquee printer-stat strip
- Featured artworks grid with hover glows and a per-piece detail dialog
- Printer spec showcase (Creality K2 Plus)
- Viral-drops teaser block (GitHub link wiring ready for Phase 2)
- Full **Gallery** page wired to the CMS
- **Payload Studio** at `/admin` with `Artworks`, `ViralPicks`, `Media`, `Users` collections
- WhatsApp + Instagram CTAs on every piece (deep-linked with pre-filled messages)

## Getting started

```bash
# 1. Install
npm install

# 2. Set local env (.env is already prefilled with dev defaults; edit to taste)
#    - PAYLOAD_SECRET                  rotate before deploy
#    - DATABASE_URI                    defaults to a local SQLite file
#    - NEXT_PUBLIC_WHATSAPP_NUMBER     your number, no +, no spaces (e.g. 919876543210)
#    - NEXT_PUBLIC_INSTAGRAM_HANDLE    handle without the @
#    - NEXT_PUBLIC_SITE_NAME           display name

# 3. Run the dev server (Next + Payload share one process)
npm run dev

# 4. Open
#    - Frontend:  http://localhost:3000
#    - Studio:    http://localhost:3000/admin
```

The first time you hit `/admin`, Payload will run migrations against the SQLite file and prompt you to create the first admin user.

## Content model

- **Artworks** — title, slug, tagline, rich description, category, cover image, gallery, materials, print time, accent color, WhatsApp message override, `featured` flag.
- **ViralPicks** — title, pitch, cover, GitHub URL, tags, order.
- **Media** — images & videos, with `thumb / card / hero` derivatives generated by Sharp.
- **Users** — admin auth.

Before you add real content, the public pages render polished demo placeholders so the site never looks empty.

## Deploy: Vercel + Neon

The database adapter auto-switches on the `DATABASE_URI` scheme — `file:` uses SQLite (local dev), `postgres://` uses Neon/Postgres (production). No code change needed between environments. **Vercel's filesystem is ephemeral, so production must use Neon** (SQLite would reset on every deploy).

1. **Create a Neon database** at [neon.com](https://neon.com) and copy the pooled connection string (looks like `postgres://user:pass@ep-xxx.aws.neon.tech/neondb?sslmode=require`).
2. **Import the repo into Vercel** (Framework preset: Next.js — auto-detected).
3. **Set Environment Variables** in the Vercel project (Production + Preview):
   - `DATABASE_URI` → your Neon pooled connection string
   - `PAYLOAD_SECRET` → a long random string (e.g. `openssl rand -base64 32`)
   - `NEXT_PUBLIC_WHATSAPP_NUMBER`, `NEXT_PUBLIC_INSTAGRAM_HANDLE`, `NEXT_PUBLIC_SITE_NAME`
4. **Deploy.** On first boot, Payload creates its tables in Neon; visit `/admin` to create your admin user.

### One thing to wire before going live: media storage

Uploaded images/videos currently save to the local `/media` folder, which also won't survive on Vercel. Before relying on uploads in production, add a storage adapter — e.g. `@payloadcms/storage-vercel-blob` (simplest on Vercel) or `@payloadcms/storage-s3` (Cloudflare R2 / AWS S3). This is a small follow-up; flagged in the roadmap below.
